# Scope Guard

## Allowed This Cycle

- Track 1: Home hero revamp
- Track 2: Global theme/layout consistency revamp

## Explicitly Excluded

- Track 3 features and experiments
- RPG status board related work
- Any large redesign outside hero/theme goals

## Change Discipline

- Keep edits narrow and intentional.
- Avoid opportunistic refactors.
- Respect current architecture and API shape unless change is required.
- Prefer file-scoped commits with clear intent.

## Escalation Rule

If a requested task requires crossing scope boundaries, pause and ask for explicit approval before proceeding.
