# First Week Guide

> **Version**: 1.0.0  
> **Last Updated**: 2025-11-17  
> **Status**: Active

This guide outlines suggested tasks and goals for your first week on the backend team.

---

## Day 1-2: Orientation & Fundamentals

- [ ] Complete [First-Day-Guide](./First-Day-Guide.md)
- [ ] Run backend locally & verify health endpoint
- [ ] Read key docs:
  - `backend/docs/README.md`
  - `backend/docs/ARCHITECTURE/README.md`
  - `backend/docs/PATTERNS/README.md`
- [ ] Review open issues & project board
- [ ] Pair with mentor/buddy for code walkthrough

---

## Day 3-4: Hands-On Contributions

- [ ] Fix a doc issue or small bug (â‰¤ 200 LOC)
- [ ] Add/Improve a unit test for existing service
- [ ] Explore admin API via Postman / Thunder Client
- [ ] Attend code review session (comment or observe)
- [ ] Shadow deployment or CI pipeline (optional)

Suggested starter tasks:
- Update DTO validation messages
- Add sample data seeding script
- Improve logging/error handling in a controller

---

## Day 5: Independent Work

- [ ] Pick a low/medium complexity ticket with mentor approval
- [ ] Draft design notes (if needed) & get feedback
- [ ] Implement feature/test/docs updates
- [ ] Open PR & request review
- [ ] Share progress in stand-up / Slack

---

## Week-End Review

- [ ] Demo what you worked on to team/mentor
- [ ] List questions/pain points encountered
- [ ] Schedule follow-up 1:1 if needed
- [ ] Update onboarding docs if you found gaps

---

## Next Steps

- Tackle medium-sized feature
- Own part of documentation or testing
- Participate in on-call rotation training (when ready)

---

**Need help?** Reach out in `#backend-help`, ping your mentor, or schedule pairing.
