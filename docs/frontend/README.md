## Documentation Principles

All documentation follows these principles:

- **Clear**: Use easy-to-understand language
- **Concise**: Include only necessary information
- **Structured**: Use scannable structure, headings, and table of contents
- **Rationale-Driven**: Include "why" for all design decisions
- **Continuity**: Sustainable documentation structure for future developers

## Contributing

To improve or add new documentation:

1. Ensure the document fits the purpose of the appropriate category
2. Follow Clear, Concise, Structured principles
3. Code examples should follow the project's patterns
